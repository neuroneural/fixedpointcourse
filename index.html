Fixed Points, Feedback,
and Adaptive Computation

Replacing fixed-depth ballistic computation with iterative, stateful processes — so that models can allocate effort, check intermediate results, and generalise as processes rather than fixed input–output maps.

A 6-lecture graduate course  ·  Sergey Plis

Course Lectures

  • L1
    Architectures, Recurrence, and Automatic Differentiation
    FFNNs, RNNs, Transformers as fixed-depth graphs. RNNs as dynamical systems. Forward & reverse-mode automatic differentiation.
    →
  • L2
    Fixed-Point Computation and Implicit Differentiation
    Banach fixed-point theorem, contraction mappings, implicit function theorem. Almeida–Pineda / RBP through modern Deep Equilibrium Models.
    →
  • L3
    Adaptive Computation: Halting, Routing, and Dynamic Depth
    ACT, Universal Transformers, Depth-Adaptive Transformers, Mixture-of-Depths — learning how much computation to spend per input or token.
    →
  • L4
    Iterative Refinement and Introspective Computation
    Networks that look at their own internal state and refine it — Slot Attention, MIND, SELF-Transformer (FPSA), Continuous Thought Machines.
    →
  • L5
    Memory, State Tracking, and Long-Horizon Reasoning
    Neural Turing Machines, Titans (test-time memorisation), ultra-long context, and inductive biases of RNNs vs. Transformers for state tracking.
    →
  • L6
    Test-Time Learning, Self-Modification, and Nested Learners
    MAML, TTT layers, Nested Learning — models that learn their own update rules, turning the optimisation process into a learnable, differentiable component.
    →
