Multilayer Perceptrons (MLPs)

By stacking layers of perceptrons, we create deep feed-forward networks capable of learning hierarchical representations. Features extracted in early layers are combined in deeper layers to solve complex computations.
  • Single perceptron $\implies$ Linear decision boundary.
  • Stacked layers $\implies$ Complex composition of non-linear functions $\sigma(W\vec{h} + \vec{b})$.
  • Computation flows strictly in one direction: input $\to$ hidden layers $\to$ output.

Universal Approximation

Kolmogorov–Arnold Representation Theorem (1956–57)
Hilbert's 13th problem · non-constructive $$f(x_1, \ldots, x_n) = \sum_{q=0}^{2n} \Phi_q\!\left(\sum_{p=1}^{n} \phi_{q,p}(x_p)\right)$$
Cybenko's Theorem (1989)
constructive · the approximator is an explicit neural network $$\left|\, f(\vec{x}) - \sum_{i=1}^{N} \alpha_i\,\sigma\!\left(\vec{w}_i^{\!\top}\vec{x} + b_i\right)\right| < \epsilon$$
Kolmogorov, Doklady 114 (1957) · Arnold, Doklady 114 (1957) · Cybenko, MCSS 2 (1989)

Kolmogorov–Arnold (1956–57): Hilbert's 13th problem (1900) asked whether every continuous function of several variables can be expressed via functions of fewer variables. Kolmogorov first reduced to 3 variables (1956); his 19-year-old student Arnold sharpened this to 2 (1957); Kolmogorov then proved the final form: any multivariate continuous function decomposes into univariate functions and addition. Non-constructive — the inner functions φ exist but can be pathological.

Cybenko (1989): A single hidden layer with sigmoidal activations σ is dense in C([0,1]^n): for any continuous f and any ε > 0, there exist finitely many weights such that the network approximates f within ε. Constructive form — the approximator is an explicit neural network architecture.

Connection: Neural networks are compositions of univariate activations and affine sums — precisely the structure both theorems describe. Kolmogorov–Arnold tells us the representation exists; Cybenko tells us neural networks can realize it.

The Ballistic Paradigm

A standard feed-forward network applies a fixed sequence of transformations to every input — the same depth, the same cost, regardless of whether the input is trivial or ambiguous.
  • Input $\vec{x}$ flows through layers $\ell = 1, \ldots, L$ without feedback.
  • Each layer: $\vec{h}^{(\ell)} = \sigma\!\left(W^{(\ell)}\vec{h}^{(\ell-1)} + \vec{b}^{(\ell)}\right)$
  • Depth $L$ is a design choice, not adapted to the input.
  • Universal approximation: in principle, any continuous function.

Why Call It "Ballistic"?

Like a projectile once fired, the computation cannot check, correct, or adapt its own intermediate results. There is no iteration, no self-reference, no state beyond the current activation.
  • Gradient computed via backprop — training is iterative, inference is not.
  • This is the paradigm that the rest of this course challenges.

Back Propagation Through Time

Unroll over time $t=1\dots T$ and apply chain rule to weight $\vec{w}$: \begin{align} \frac{\partial L}{\partial \vec{w}} &= \frac{\partial L}{\partial \vec{h}_T} \sum_{k=1}^T \frac{\partial \vec{h}_T}{\partial \vec{h}_{T-k+1}} \frac{\partial \vec{h}_{T-k+1}}{\partial \vec{w}} \\ &= \frac{\partial L}{\partial \vec{h}_T} \sum_{k=1}^T \left( \prod_{i=T-k+2}^{T} J_{F, \vec{h}}(\vec{x}, \vec{w}, \vec{h}^{i-1}) \right) \frac{\partial F}{\partial \vec{w}} \end{align}
TBPTT (Truncated): We typically unroll only $t_0$ steps to avoid unbounded memory and vanishing gradients. We will dive deeper next lecture!

Sequence-to-Sequence (Seq2Seq)

Map a sequence $\vec{x}_{1:T_x}$ to $\vec{y}_{1:T_y}$ using
an Encoder and a Decoder.
  • Encoder: Processes input to a fixed-length context vector $\vec{c} = \vec{h}_{T_x}$.
  • Decoder: Generates output conditioned on $\vec{c}$ and previous outputs.
  • The Bottleneck: The entire input must be compressed into a single vector $\vec{c}$.
Sutskever, I., Vinyals, O. and Le, Q.V., 2014. Sequence to sequence learning with neural networks. Advances in neural information processing systems, 27.

Attention

Instead of one static context vector $\vec{c}$, let the Decoder "attend" to all Encoder states dynamically at each step.
For decoder hidden state $\vec{s}_i$, compute alignment scores with all encoder states $\vec{h}_j$:
$$ e_{ij} = a(\vec{s}_{i-1}, \vec{h}_j) $$
Convert to attention weights via softmax:
$$ \alpha_{ij} = \frac{\exp(e_{ij})}{\sum_k \exp(e_{ik})} $$
Compute dynamic context vector as weighted sum:
$$ \vec{c}_i = \sum_j \alpha_{ij} \vec{h}_j $$
Bahdanau, Cho, Bengio: "Neural Machine Translation by Jointly Learning to Align and Translate" (2014)

Dot-Product Attention

Same attention pipeline, different alignment score. Luong et al. compare dot, general, and concat scoring functions.
Dot score: $$e_{ij}=\vec{s}_i^\top\vec{h}_j$$
Luong alternatives general and concat are shown in the diagram.
$$\alpha_{ij}=\operatorname{softmax}_j(e_{ij}),\qquad \vec{c}_i=\sum_j\alpha_{ij}\vec{h}_j$$
Dot-product is parameter-light and efficient, while preserving the same context-vector computation.
Luong, M.-T., Pham, H., & Manning, C. D. (2015). Effective approaches to attention-based neural machine translation. In EMNLP 2015 (pp. 1412-1421).

A procedure for automatic evaluation of total/partial derivatives of arbitrary algebraic functions is presented. What is Automatic Differentiation?

Automatic Differentiation (AD) mechanically calculates the derivatives (Leibnitz, 1664; Newton, 1704) of functions expressed as computer programs, at machine precision, and with complexity guarantees.

Automatic Differentiation

  • Derivative of $f: \RR^n \to \RR^m$ is $m\times n$ "Jacobian matrix" $\bm{J}$.
  • AD in the forward accumulation mode: $\bm{J}v$ (Wengert, 1964)
  • AD in the reverse accumulation mode, $\bm{J}^Tv$ (Speelpenning, 1980)
  • About a zillion other modes and tricks
  • Vibrant field with regular workshops, conferences, updated community portal (http://autodiff.org)

What is wrong with Numerical Differentiation

This plot shows the fundamental catch-22 of numerical differentiation — a log-log plot of error vs. step size h.

Truncation Error (Right side): When h is large, our finite difference approximation is inaccurate mathematically. We dropped (truncated) higher-order terms from the infinite Taylor series. The dashed blue line (center difference) gets steeper because it is $O(h^2)$, compared to the $O(h)$ forward difference.

Round-off Error (Left side): To fix truncation error, we shrink h. But as h gets tiny, f(x+h) and f(x) become almost identical in the computer's memory. Subtracting them causes "catastrophic cancellation", destroying all significant digits and leaving pure machine noise. Dividing that noise by a tiny h makes the error explode upward.

The Punchline: We are trapped inside that V-shape. We can never take the mathematical limit of $h \to 0$ safely on a computer. Automatic differentiation entirely bypasses this by calculating exact analytic derivatives programmatically.

Dual numbers (1873)

\begin{align} \fragment{1}{v + \dot{v}\epsilon} &\\ \fragment{2}{v, \dot{v} \in \RR,} & \fragment{2}{\epsilon \ne 0, \epsilon^2 = 0} \\ \fragment{3}{(v + \dot{v}\epsilon) + (u + \dot{u}\epsilon)} & \fragment{3}{= (v + u) + (\dot{v} + \dot{u})\epsilon}\\ \fragment{4}{(v + \dot{v}\epsilon)(u + \dot{u}\epsilon)} & \fragment{4}{= (vu) + (v\dot{u} + u\dot{v})\epsilon}\\ \fragment{5}{\frac{v + \dot{v}\epsilon}{u + \dot{u}\epsilon}} & \fragment{5}{= \frac{v}{u} + \frac{u\dot{v} - v\dot{u}}{u^2}\epsilon}\\ \fragment{6}{f(v+\dot{v}\epsilon)} & \fragment{6}{= f(v) + f^{\prime}(v)\dot{v}\epsilon}\\ \fragment{7}{f(g(v+\dot{v}\epsilon))} & \fragment{7}{= f(g(v)) + f^{\prime}(g(v))g^{\prime}(v)\dot{v}\epsilon}\\ \fragment{8}{v + \dot{v}\epsilon} & \fragment{8}{\cong \begin{pmatrix} v & \dot{v} \\ 0 & v \end{pmatrix}} \end{align}

Notes on exactly how Dual Number AD works natively:

1. How does the fundamental rule f(a + bε) = f(a) + f'(a)bε emerge?
It evaluates strictly from the standard Taylor series! Take the expansion of $f(a+h)$: $f(a) + f'(a)h + \frac{1}{2!}f''(a)h^2 + \dots$
Substitute $h = b\epsilon$. Because we postulate $\epsilon^2 = 0$, every single term with $\epsilon^2, \epsilon^3$, etc., vanishes algebraically to exactly zero. You are left purely with $f(a) + f'(a)b\epsilon$.

2. How does the Division (Quotient) Rule naturally emerge?
Exactly like complex number division—you multiply the numerator and denominator by the "conjugate" $u - \dot{u}\epsilon$!
$$ \frac{v + \dot{v}\epsilon}{u + \dot{u}\epsilon} \cdot \frac{u - \dot{u}\epsilon}{u - \dot{u}\epsilon} $$
Denominator: expands to $u^2 - u\dot{u}\epsilon + u\dot{u}\epsilon - (\dot{u}\epsilon)^2$. Since $\epsilon^2 = 0$, the denominator collapses exactly to $u^2$.
Numerator: expands to $vu - v\dot{u}\epsilon + u\dot{v}\epsilon - \dot{v}\dot{u}\epsilon^2$. Again $\epsilon^2 = 0$, so it collapses to $vu + (u\dot{v} - v\dot{u})\epsilon$.
Bringing them together: $\frac{vu}{u^2} + \frac{(u\dot{v} - v\dot{u})\epsilon}{u^2} = \frac{v}{u} + \frac{u\dot{v} - v\dot{u}}{u^2}\epsilon$.
Behold: exactly the quotient rule!

3. How does the Chain Rule emerge?
Apply the fundamental rule to the inner function: $g(v+\dot{v}\epsilon) = g(v) + g'(v)\dot{v}\epsilon$.
Treating $g(v)$ as your new real part '$A$' and $g'(v)\dot{v}$ as your new dual part '$B$', plug this directly into $f(A + B\epsilon)$.
It expands to $f(A) + f'(A)B\epsilon$. Substituting back, we receive $f(g(v)) + f'(g(v))\cdot g'(v)\dot{v}\epsilon$.
The coefficient attached to $\epsilon$ is strictly the chain rule multiplying our initial seed!

4. Why the Matrix Isomorphism? How does it show the Sum and Product Rules?
If you represent a dual number $v + \dot{v}\epsilon$ as a 2x2 matrix $\begin{pmatrix} v & \dot{v} \\ 0 & v \end{pmatrix}$, standard linear algebra flawlessly simulates AD!
Sum Rule: Add two dual matrices together. You get $\begin{pmatrix} u+v & \dot{u}+\dot{v} \\ 0 & u+v \end{pmatrix}$. The upper right is naturally $\dot{u}+\dot{v}$.
Product Rule: Multiply two matrices $\begin{pmatrix} v & \dot{v} \\ 0 & v \end{pmatrix} \begin{pmatrix} u & \dot{u} \\ 0 & u \end{pmatrix} = \begin{pmatrix} vu & v\dot{u} + u\dot{v} \\ 0 & vu \end{pmatrix}$. The upper right is EXACTLY the derivative product rule $v\dot{u} + u\dot{v}$!

Speaker Notes: "The Classical Presentation vs Reverse AD"

The standard Backprop story (the "ugly" version):
When backprop was popularized for neural networks in the 1980s, the community explained it purely in terms of "neurons" and "layers." Students are forced to stare at monstrous equations full of nested summations, indices ($i, j, k$), and bespoke "delta rules" for hidden vs. output layers. It feels like a highly specialized, incredibly fragile algorithm that only works if your math strictly aligns with multi-layer perceptrons.

The Reverse Mode AD story (the "elegant" version):
Reverse Mode AD comes from the compiler community, and it steps back and says: Forget neurons. Everything is just a computation graph.

Instead of wrestling with global chain rule summations, we recognize a beautiful local principle: every intermediate variable just receives "sensitivities" (adjoints, denoted $\bar{v}$) from the nodes it pushes data to, multiplies that by the local derivative, and passes it backward. It replaces complex matrix-calculus proofs with simple, mechanical local message passing on a graph.

The Takeaway: By viewing backprop simply as Reverse-Mode AD on a computation graph, the mystery vanishes. You don’t need a different rule for a CNN, an RNN, or an arbitrary physics simulation. As long as you can draw the forward graph, the reverse graph writes itself.

Reverse Mode AD

In the 1970s, tools for automated generation of adjoint codes (aka reverse accumulation mode automatic differentiation, aka backpropagation) were developed.
Type I: Pure Mathematics. Brilliant derivations of the adjoint state method for specific continuous systems. It required a human genius doing pure calculus, not automated algorithms.
Type II: Hand-Coded Gradients. The manual derivation and hard-coding of reverse-mode difference equations (like old-school neural network backprop). It was tedious and incredibly topology-specific.
Type III: Source-to-Source Compilers. A specialized compiler deeply analyzes your source code's AST and explicitly writes thousands of lines of new source code representing the exact derivative.
Type IV: Differentiable Programming. The modern standard. Gradients are natively generated dynamically on-the-fly with first-class AD operators implicitly integrated into the language or framework.
Type I
Geniuses transforming mathematical systems
(Gauss; Feynman (1939); Rozonoer and Pontryagin (1959))

Speaker Notes: Where does TensorFlow live in this hierarchy?

TensorFlow actually underwent a massive paradigm shift that perfectly tells this historical story!

1. TensorFlow 1.x is fundamentally a modern flavor of Type III:
In TF v1, you didn't evaluate standard code; you built a static, symbolic computation graph. When you called an optimizer, TensorFlow took your graph and acted exactly like a Graph-to-Graph compiler. It analyzed your forward graph and statically embedded hundreds of new symbolic gradient nodes into it before anything actually executed. It was rigid and clunky, but highly optimized—classic Type III.

2. TensorFlow 2.x (along with PyTorch) operates squarely in Type IV:
When TF v2 launched with Eager Execution and tf.GradientTape, it abandoned the static graph and fully crossed the bridge into Type IV (Differentiable Programming). Now, you execute standard Python dynamically, and the "tape" silently traces the operations via operator overloading and closures under the hood. The derivative evaluates on-the-fly automatically alongside your standard runtime code.

The JAX Hybrid:
JAX is the ultimate chimera. In its API, it acts beautifully like Type IV (first class functions, jax.grad closures). But completely under the hood, it rigidly traces python down to "jaxpr" expressions and hands them to XLA, acting exactly like a hyper-optimized Type III source-to-source compiler!

Two Directions, Three Traces

\begin{align} y &= \log(\sin(x^2)) \end{align}
  • Traces:
  • Primal
  • Tangent Derivative
  • Cotangent Derivative
  • Direction:
  • $\rightarrow$ Forward
  • $\rightarrow$ Forward
  • $\leftarrow$ Reverse

Every AD computation involves three conceptually distinct passes through the same function. Use this example — y = log(sin(x²)) — to make it concrete.

The Primal trace is just ordinary execution: plug in a number for x, compute x², compute sin of that, compute log of that. You get a number out. This flows strictly forward, left to right through the graph. Nothing special here — every program does this.

The Tangent derivative trace is forward mode AD. You carry a second "shadow" value alongside each intermediate — the derivative of that intermediate with respect to your chosen input direction. This also flows forward, in lockstep with the primal. One forward pass gives you one column of the Jacobian — one directional derivative. Efficient when you have few inputs, many outputs.

The Cotangent derivative trace is reverse mode AD — backpropagation. You first run the primal forward to record the computation graph, then you flow sensitivity information backward from the output to every input. One reverse pass gives you one row of the Jacobian — the full gradient with respect to all inputs at once. This is why backprop is so powerful for neural networks: you have many parameters, one scalar loss.

The punchline: forward and reverse mode are duals of each other — one gives you Jv (Jacobian-vector product), the other gives you JTv (vector-Jacobian product). Both are exact, at machine precision. The choice between them is purely a question of whether your function is "tall" (more outputs than inputs → prefer forward) or "wide" (more inputs than outputs → prefer reverse).

Note: Why "Tangent" and "Cotangent"? (Geometric Intuition)
Tangent (Forward): Imagine you pick a direction to move. That velocity vector geometrically lives in the Tangent Space. Because forward-mode AD pushes a perturbation direction forward through the graph, it calculates a Tangent Derivative.
Cotangent (Reverse): In linear algebra, the "dual" to a vector space is the set of linear operations that map vectors to scalars. Geometrically, the dual to the Tangent Space is the Cotangent Space. Reverse-mode AD starts with a final scalar (like Loss) and pulls sensitivities backward. Because these gradients act exactly as linear operators waiting to eat a tangent perturbation, they fundamentally live in the Cotangent Space! Hence, backprop calculates Cotangent Derivatives (geometric pullbacks).

\begin{align} x &\\ z_1 &= x^2\\ z_2 &= \sin(z_1)\\ z_3 &= \log(z_2)\\ y &= z_3\\ \end{align}
  • $\bar{x} = \left(\left(\frac{\partial y}{\partial z_3}\frac{\partial z_3}{\partial z_2}\right) \frac{\partial z_2}{\partial l z_1}\right)\frac{\partial z_1}{\partial x}$
  • $\bar{x} = \frac{\partial y}{\partial x} = \frac{\partial y}{\partial z_1}\frac{\partial z_1}{\partial x} = \bar{z}_1 2 x$
  • $\bar{z}_1 = \frac{\partial y}{\partial z_1} = \frac{\partial y}{\partial z_2}\frac{\partial z_2}{\partial z_1} = \bar{z}_ 2\mathrm{cos}(z_1)$
  • $\bar{z}_2 = \frac{\partial y}{\partial z_2} = \frac{\partial y}{\partial z_3}\frac{\partial z_3}{\partial z_2} = \bar{y}\ \frac{1}{z_2}$
  • $\bar{z}_3 = \frac{\partial y}{\partial z_3} = \frac{\partial y}{\partial y} = \bar{y} = 1$ (seed)

Takeaways

  • FFNNs & Transformers are "ballistic" — fixed depth, fixed cost per input; the paradigm this course challenges.
  • RNNs are dynamical systems — hidden state as lossy summary; convergent RNN → fixed point of $f_\theta$ (L2).
  • Attention is a higher-order, data-dependent operator — $\mathcal{T}$ computed on the fly, previewing L4 / L5 / L6.
  • Automatic differentiation — exact $\mathbf{J}v$ / $\mathbf{J}^\top v$ for any computation graph; the tool for what comes next.
