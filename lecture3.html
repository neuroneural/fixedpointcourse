Why Should Computation Be Adaptive?

The Fixed-Depth Problem

Hard input
Layer 1

Layer 2

A 96-layer Transformer spends the same compute predicting "the" as it does predicting a rare technical term.

This is the core tension: feed-forward networks are "ballistic" — once the input enters, it passes through every layer regardless of difficulty. RNNs at least have variable-length sequences, but each step uses the same computation. What if we could vary the amount of computation per input?

Human Brains Are Not Fixed-Depth

  • Multi-circuit neurons: the same cortical neurons participate in multiple distinct functional circuits and intrinsic networks — neural "weight sharing" across tasks
    Sporns (2011); Bassett & Sporns (2017)
  • Bypass mechanisms: some stimuli trigger rapid responses through shortcut pathways (amygdala fast pathway for threat), while complex stimuli require multi-stage cortical processing
    LeDoux (1996); Pessoa & Adolphs (2010)
  • Effort scales with difficulty: fMRI shows that harder reasoning tasks recruit more widespread cortical activation and sustain it longer
    Duncan (2010); Fedorenko et al. (2013)

Multi-circuit neurons: A single neuron in prefrontal cortex may participate in working memory circuits, decision-making circuits, and attention circuits — depending on the current task demand. This is the biological analogue of weight sharing in Universal Transformers.

Bypass: The classic example is LeDoux's "low road" — the thalamus-to-amygdala shortcut that enables fast fear responses. The "high road" goes through cortex for full processing. The brain adaptively routes information through different-depth pathways.

Effort: The multiple demand network (Duncan 2010) activates more strongly and broadly for harder cognitive tasks. This is the neural correlate of adaptive computation — the brain allocates more neural "compute" to harder problems.

Neural Multiplexing

  • The same neurons engage in different circuits at different times — forming task-specific functional networks dynamically
  • This is remarkably similar to weight-shared architectures (Universal Transformer, RBP) where the same parameters are reused across "depth"
  • Unlike fixed feed-forward nets, the brain does not dedicate separate hardware to each processing stage — it recycles
"The brain's 'layers' are not a pipeline — they are a recurrent, adaptive, depth-varying computation."
This lecture: architectures that bring this flexibility to neural networks.

Bridge from neuroscience to the lecture content. The key analogy: the brain reuses the same neural populations across multiple processing stages and adjusts how many stages to run. We're now going to see exactly this in neural network architectures.

The weight-sharing connection is literal: Universal Transformers reuse the same layer parameters, just as cortical columns reuse the same neural populations across different functional networks.

Adaptive Computation Time

Learning When to Stop

Graves, "Adaptive Computation Time for Recurrent Neural Networks" (2016)

The Core Idea

An RNN that applies the same transition function multiple times per input, with a learned halting probability at each "ponder" step.
  • At each time step $t$, the network iterates internally for $N(t)$ sub-steps
  • $N(t)$ is not fixed — it is determined by a learned halting unit
  • The network is unrolled in depth, not just in time

This is the fundamental diagram — show how the RNN processes each input token with a variable number of internal computation steps. Easy tokens might take 1–2 steps, hard tokens might take 5–7. The key insight is that the number of steps is learned, not hand-set.

Point out the two axes in the diagram: horizontal = time steps (sequence), vertical = ponder steps (depth). Standard RNNs have depth 1 at each time step. ACT makes depth variable and learnable.

The Halting Mechanism

At ponder step $n$ of time step $t$, compute a halting probability:
$$h_t^n = \sigma\!\left(W_h \, s_t^n + b_h\right)$$
Accumulate into a running sum:
$$H_t^n = \sum_{k=1}^{n} h_t^k$$
Define the total steps $N(t)$ as the first step $n$ where $H_t^{n} \geq 1 - \epsilon$, and halt.
Compute the remainder: $\quad R_t = 1 - \sum_{k=1}^{N(t)-1} h_t^k$
Deterministic + differentiable: no sampling, no REINFORCE — just a sigmoid and an accumulator.

Walk through the mechanism step by step. Key points:

1. h is a sigmoid — bounded in (0,1). Each ponder step contributes a "probability mass."

2. The halting probabilities accumulate. Once they sum to ~1, the network stops.

3. The remainder R handles the edge: the last step gets whatever probability mass is left.

4. This is fully differentiable — gradients flow through the sigmoid. No need for REINFORCE or Gumbel tricks.

Weighted Output

The final state is a weighted combination of all ponder states:
$$\bar{s}_t = \sum_{n=1}^{N(t)} p_t^n \cdot s_t^n$$
where the weights are:
$$p_t^n = \begin{cases} h_t^n & \text{if } n \lt N(t) \\ R_t & \text{if } n = N(t) \end{cases}$$
The network doesn't just decide when to stop — it interpolates over its intermediate states.
The halting weights $p_t^n$ sum to 1 by construction.

This is a subtle but important point. The output isn't just the last ponder state — it's a weighted average. Early ponder states contribute if they had high halting probability. This means the network can "hedge" — partially committing to early answers while continuing to refine.

The weights sum to 1 by construction: the first N-1 terms are the raw halting probabilities, and the last term R is defined as the remainder 1 - sum.

The Ponder Cost

Without a penalty, the network would always ponder maximally.
$$\mathcal{L} = \mathcal{L}_{\text{task}} + \tau \cdot \rho(t)$$
where the ponder cost is: $$\rho(t) = N(t) + R_t$$
  • $\tau$ is a hyperparameter trading accuracy for compute
  • Small $\tau$ → more pondering, higher accuracy
  • Large $\tau$ → fewer steps, faster but less accurate
  • This is a surrogate for a computational budget — not a hard constraint

The ponder cost is key to making ACT practical. Without it, the network would always use the maximum number of steps — more computation never hurts task loss. The penalty encourages the network to stop early when it can.

Note that ρ(t) = N(t) + R is essentially "number of steps plus fractional remainder." The remainder R makes this differentiable w.r.t. the halting probabilities.

The τ hyperparameter is both ACT's strength and weakness: it gives direct control over the compute-accuracy tradeoff, but it requires tuning and may not generalize across tasks or data distributions.

ACT Pseudocode


def ACT_step(x_t, s_prev, W_h, b_h, τ_max):
    """One time step of an ACT-augmented RNN."""
    H = 0.0              # cumulative halting probability
    s_bar = 0             # weighted output state
    n = 0

    while H < 1 - ε:
        n += 1
        s_n = RNN_cell(x_t, s_prev if n == 1 else s_n)
        h_n = σ(W_h @ s_n + b_h)     # halting probability

        if H + h_n ≥ 1 - ε or n == τ_max:   # halt
            R = 1 - H                 # remainder
            s_bar += R * s_n
            break
        else:
            H += h_n
            s_bar += h_n * s_n

    ponder_cost = n + R   # for the loss penalty
    return s_bar, ponder_cost

Walk through the pseudocode step by step. Emphasize that the while loop IS the "pondering" — the network iterates internally before moving to the next input token. The halting probability h_n is computed fresh at each ponder step from the current state s_n.

Note that s_prev is the hidden state from the previous time step (carried across time), while s_n is the state within the current ponder loop (carried across depth).

Synthetic Tasks: Ponder Time Adapts

  • Parity, logic, integer addition, sorting — ACT dramatically outperforms fixed-depth
  • Key observation: ponder time scales with input difficulty
  • Longer bit strings → more ponder steps; harder sorting instances → more steps
Graves, "Adaptive Computation Time for Recurrent Neural Networks" (2016)

These are the "clean" results. On synthetic algorithmic tasks, ACT learns to use more computation for harder instances. This is exactly the behavior we wanted: adaptive compute allocation.

Point out specific examples: for parity, longer inputs need more steps because the parity computation requires visiting all bits. For sorting, the number of steps correlates with the number of inversions.

Language Model: Where Compute Goes

  • Ponder time spikes at spaces, punctuation, and rare transitions
  • Correlates with prediction loss and entropy — the network thinks harder when it's uncertain
  • Suggests ACT learns a form of implicit difficulty estimation
Graves (2016), Figure 23

This is the "wow" figure of the paper. The three-panel layout shows that ponder time, prediction loss, and prediction entropy are highly correlated — the network spends more computation on characters that are genuinely harder to predict.

Specific observations: spaces between words are prediction boundaries (next word could be anything), so ponder time spikes. Common words like "the" get low ponder time. Rare or surprising transitions get high ponder time.

This also suggests ACT as a tool for data analysis — the ponder map reveals the difficulty structure of the data, potentially useful for segment boundary detection.

ACT: Strengths and Limitations

Strengths

  • Simple, deterministic, differentiable
  • Interpretable ponder maps
  • Minimal architectural changes to the base RNN
  • Direct compute–accuracy tradeoff via $\tau$

Limitations

  • $\tau$ is sensitive and requires tuning
  • Halting distribution can collapse — always using ~same steps
  • No probabilistic model of halting — just a threshold
  • Ponder penalty is a crude surrogate for computational budget
Can we do better? PonderNet offers a probabilistic fix…

PonderNet: A Probabilistic Twist

Key change: model halting as a geometric-like distribution over ponder steps.
$$p(\text{halt at step } n) = \lambda_n \prod_{k=1}^{n-1}(1 - \lambda_k)$$
where $\lambda_n = \sigma(W_h s_n + b_h)$ — same halting unit as ACT.
  • Training: KL divergence between the learned halting distribution and a geometric prior
  • Replaces $\tau$ with a prior distribution — more principled regularization
  • At inference: sample or take the mode of the halting distribution
Banino et al., "PonderNet: Learning to Ponder" (2021)

PonderNet recasts ACT's halting mechanism as a proper probability distribution. Instead of accumulated probability hitting a threshold, each step has a conditional halting probability λ_n, and the probability of halting at exactly step n follows a geometric-like distribution.

The KL regularization replaces τ with a prior — typically a geometric distribution with a chosen expected number of steps. This is more principled than the raw ponder penalty.

The Thread

  • ACT (2016): deterministic threshold — "halt when accumulated probability hits 1"
  • PonderNet (2021): probabilistic model — "sample from a learned halting distribution"
  • Both: the network decides when to stop — variable compute, no hard budget
  • Coming next: what if we flip the question? Instead of "when to stop," ask "which tokens need which layers" — under a fixed budget
But first: what happens when we transplant ACT into the Transformer?

This slide connects PonderNet back to ACT and forward to routing. The key conceptual progression: ACT = threshold-based halting → PonderNet = probabilistic halting → routing methods (§4) = skip/route instead of halt. But before routing, we need to see what happens when ACT moves from RNNs to Transformers — that's the Universal Transformer.

Universal Transformer

Weight-Sharing + Halting in Attention

Dehghani et al., "Universal Transformers" (2019)

From Stacked to Shared Layers

  • UT = Transformer body applied recurrently over depth (+ step embeddings)
  • This is literally a weight-tied deep network — an RNN over depth

The key architectural difference: a standard Transformer has L different layers, each with its own parameters. The UT has ONE set of layer parameters that is applied T times. This is weight sharing across depth — the same idea as an RNN unrolled in time, but here "time" is "depth."

The UT adds coordinate/step embeddings (similar to positional embeddings) so the shared layer knows which step it's on. Without these, the shared layer would apply the same function identically at each step — with them, it can condition on the current depth.

Per-Position Adaptive Halting

  • ACT halting applied independently at each position in the sequence
  • Different tokens halt at different depths — function words early, content words later
  • The result: a ragged depth profile across the sequence
Dehghani et al., "Universal Transformers" (2019)

This is the key figure from the UT paper. Each column represents a token, each row a depth step. Darker = more computation at that step. Function words ("the," "a," "is") halt after 1–2 steps. Content words and syntactically complex positions get more steps.

Note: tokens that have halted still participate in attention with tokens that haven't — their representations are frozen but visible. This means late-halting tokens can still attend to early-halting tokens.

Turing Completeness

  • Standard Transformers are not Turing-complete — they are bounded-depth circuits
  • UT with ACT = Transformer + recurrence + conditional halting
  • Under standard computability assumptions, UTs are Turing-complete
  • Intuitively: unbounded depth + halting condition = ability to simulate any computation
A theoretical advantage, not always a practical one — but it means UTs can in principle generalize to arbitrary-length inputs, while standard Transformers cannot.

This is a theoretical result. The argument is essentially: a UT with ACT can simulate a Turing machine by using its shared layer as the transition function and ACT's halting mechanism as the halt state. A standard Transformer with fixed depth L is a bounded-depth circuit — it cannot compute functions that require more than L "steps" of reasoning.

In practice, this matters for length generalization: UTs can sometimes generalize to longer inputs than seen in training (because they can ponder longer), while standard Transformers often fail catastrophically on out-of-distribution lengths.

UT Results Highlights

  • LAMBADA (word prediction): new SOTA at publication — UT outperforms standard Transformers and LSTMs
  • Algorithmic tasks (copying, reversing): standard Transformers fail on OOD lengths, UTs generalize
  • WMT14 En-De translation: +0.9 BLEU over vanilla Transformer
  • Subject–verb agreement: UT matches or beats Transformer across all attraction distances
The win is primarily on generalization beyond training distribution — exactly what adaptive depth promises.

Key takeaway: UT's gains are most dramatic on tasks requiring length generalization or compositional reasoning — exactly where adaptive depth matters. On standard benchmarks with fixed-length inputs, the gains are more modest.

UT ↔ RBP: The Same Idea?

UT with infinite shared steps → weight-tied infinite depth → converge to a fixed point:
$$\vec{z}^* = f_\theta(\vec{z}^*, \vec{x})$$
This is a Deep Equilibrium Model (L2)!
RBP (L2)
Implicit depth: solver decides convergence
Constant memory via implicit differentiation
UT + ACT (L3)
Explicit depth: halting network decides
O(depth) memory — stores all ponder states
Same spirit, different mechanism. Both break fixed-depth — RBP via convergence, UT via learned halting.

Important bridge slide. The UT, if you remove the halting mechanism and let it run to convergence, IS a DEQ. The difference is entirely in how depth is controlled: DEQ uses a numerical solver with convergence tolerance; UT uses a learned halting mechanism with a ponder penalty. DEQ gets constant memory (implicit differentiation at the fixed point); UT pays O(depth) memory for the full computation graph but gets more direct control over the compute-accuracy tradeoff.

From Halting to Routing

Adaptive Compute Under Fixed Budgets

The Pivot

  • ACT / UT: variable total compute — the network decides how long to run
  • Problem: total FLOPs are unpredictable → hard to deploy with latency guarantees
  • New question: what if we fix the total FLOP budget but let the network choose where to spend it?
  • This reframes adaptive computation from halting to routing

This is the conceptual pivot of the lecture. ACT/UT give the network freedom to choose how much compute to use, but this makes total FLOPs unpredictable — a problem for deployment. The routing approach says: you get a fixed budget, but you decide how to allocate it across tokens and layers.

Depth-Adaptive Transformer

Each layer $\ell$ has an exit classifier that can emit a prediction:
$$\hat{y}_\ell = \text{Classifier}_\ell(h_\ell)$$
  • Unlike UT: layers are not shared — each layer is a different function
  • The model learns both different representations and when to exit
  • Exit policy options: per-sequence confidence threshold, per-token decisions, or aligned training where all classifiers produce good predictions
  • At inference: exit early when confidence is high enough
Elbayad et al., "Depth-Adaptive Transformer" (2020)

Key differences from UT: (1) layers are NOT shared — different parameters at every layer, so each layer can learn different things. (2) The "halting" mechanism is different — it's an exit classifier at each layer, not an accumulated halting probability. (3) Training requires aligning all intermediate classifiers to produce good predictions — otherwise early exit gives garbage.

This is closer to "early exit" than "adaptive computation time" — the network doesn't iterate the same function, it exits a pipeline early.

DAT: Results

  • IWSLT German–English: matches baseline Transformer accuracy
  • Uses less than 25% of decoder layers on average
  • Most tokens exit very early — only a few need full depth
Elbayad et al. (2020)

The striking result: you can get the same accuracy with a fraction of the computation. Most decoder tokens are "easy" — they can be predicted after just 1–2 layers. Only a small fraction of tokens need the full depth of the network.

This motivates the next question: if most tokens don't need full computation, why not explicitly route them around the expensive layers?

Mixture-of-Depths

Fix the total compute budget. Let the network learn which tokens pass through each layer.
● Each layer has capacity of $k$ tokens (set a priori)
● A lightweight router scores all tokens: $r_i = W_r \cdot h_i$
● Only top-$k$ tokens pass through self-attention + MLP
● Rest skip via residual: $h_i^{\ell+1} = h_i^{\ell}$
Raposo et al., "Mixture-of-Depths" (2024)

The MoD mechanism is elegant in its simplicity: at each layer, a router produces a scalar score for each token. The top-k tokens go through the full attention + MLP computation. The rest just pass through unchanged via the residual connection.

Key subtleties: k is set before training — the compute budget is hard-coded. But WHICH tokens get the budget is entirely learned. Tokens that skip a layer still contribute their representation to later layers via the residual. The compute graph has fixed tensor sizes (important for hardware efficiency).

MoD: The Key Insight

"The compute budget is fixed, but its allocation is learned."
  • Unlike ACT: no halting penalty — the budget constraint is architectural
  • Unlike DAT: the decision is per-layer, per-token, not just early exit
  • Tokens that skip a layer still contribute their residual — they aren't dropped
  • Static compute graph + known tensor sizes → hardware-friendly
Think of it as: "Mixture-of-Experts applied to layers rather than sub-networks."

The MoE analogy is precise: in MoE, a router selects which expert sub-network processes a token (adaptive width). In MoD, a router selects which LAYERS process a token (adaptive depth). Both are forms of conditional computation with learned routing.

The hardware-friendliness is a practical advantage: because k is fixed, all tensor sizes are known at compile time, enabling efficient batching and parallelization. ACT and UT have variable compute per token, which creates load-balancing issues on GPUs.

MoD: Results

  • Matches baseline performance with a fraction of FLOPs per forward pass
  • Up to 50% faster at inference (post-training sampling)
  • Later layers are more selective — "easy" tokens are consistently routed around
  • Learns sensible routing without explicit difficulty labels
Raposo et al. (2024)

Key results: (1) MoD models match isoFLOP baselines in terms of training performance. (2) At inference, they're much faster because many tokens skip layers. (3) The routing patterns are interpretable — later layers tend to be more selective, and certain token types are consistently routed around at many layers.

Adaptive Depth vs. Adaptive Width

Adaptive Depth
Choose how many layers each token passes through.
ACT · UT · DAT · MoD
Adaptive Width
Choose which sub-network (expert) processes each token.
Mixture-of-Experts (MoE)
  • These are complementary axes of the same idea: allocate compute where needed
  • MoD ≈ "MoE applied to layers rather than sub-networks"
  • They can be composed — adaptive depth and width simultaneously
Shazeer et al., "Outrageously Large Neural Networks: The Sparsely-Gated Mixture-of-Experts Layer" (2017)

The MoE reference is optional reading, but the conceptual connection is important. MoE routes tokens to different expert sub-networks (adaptive width). MoD routes tokens to different layers (adaptive depth). Both use learned routers. You can combine them: some layers use MoE experts AND some tokens skip those layers entirely.

The RBP Connection (Revisited)

  • RBP (L2): implicit adaptive depth — iterate until convergence to $\vec{z}^* = f(\vec{z}^*, \vec{x})$
    ✓ Constant memory   ✓ No halting hyperparameter   ✗ Needs convergence guarantees
  • ACT / UT: explicit adaptive depth — learn a halting distribution
    ✗ O(depth) memory   ✗ Tunable τ   ✓ Direct compute–accuracy control
  • MoD: explicit routing — fixed budget, dynamic allocation
    ✓ Predictable FLOPs   ✓ Hardware-friendly   ✗ No convergence guarantees
All three break the fixed-depth paradigm — each with a different tradeoff between guarantees, control, and efficiency.

This is the synthesis slide. Emphasize that all these methods are attacking the same problem (fixed-depth computation) from different angles. DEQs are the most mathematically principled (convergence guarantees from contraction theory in L2). ACT/UT give the most direct control. MoD is the most practical for deployment.

Bridge to L4: Confidence and Introspection

  • CALM (Schuster et al., 2022): early-exit inference for LLMs
    The network estimates its own confidence to decide whether to exit early
  • But confidence estimation requires the network to introspect — to model its own uncertainty
  • This is exactly L4's theme: networks that look at their own internal states to decide what to compute next
  • L4 goes further: MIND and SELF-Transformer don't just decide whether to keep computing — they decide what to compute
Schuster et al., "Confident Adaptive Language Modeling" (CALM, 2022)

CALM is the optional reading that bridges L3 and L4 most directly. The key insight: early exit requires the network to know when it's "confident enough." But calibrating confidence is itself a hard problem — networks are often overconfident. L4 formalizes this introspective capability as an architectural principle.

Takeaways

  • Adaptive depth replaces fixed compute — ACT/UT learn per-input halting; the network decides when to stop.
  • Routing flips the question — DAT and MoD ask which tokens need which layers under a fixed FLOP budget.
  • Implicit vs explicit depth control — RBP (L2) converges to equilibrium; ACT/UT learn a halting signal.
  • Depth and width are orthogonal axes — adaptive depth composes with adaptive width (MoE), and L4 adds introspection.

Open problems to discuss if time permits:

  • Calibration: learned halting/routing can collapse to near-fixed step counts.
  • OOD robustness: halting signals trained on easy data can halt too early out of distribution.
  • Training stability: ponder penalties and routing objectives are brittle.
  • Compositional adaptivity: jointly adaptive depth and width is still underexplored.

Bridge to Lecture 4: introspection-driven compute allocation (Slot Attention, MIND, SELF-Transformer, CTM).
